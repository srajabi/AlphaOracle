# Multi-Account Trading System

## Overview

The AlphaOracle system now supports **5 separate Alpaca accounts**, each running a different strategy:

1. **Account 1** - LLM Recommendations (auto-scaled)
2. **Account 2** - Reddit 200 SMA on TQQQ
3. **Account 3** - Dual Momentum (SPY/QQQ/GLD)
4. **Account 4** - Reserved
5. **Account 5** - Reserved

Each account operates independently with its own API keys and can be enabled/disabled individually.

### Account Configuration: `config/accounts.json`

```json
{
  "accounts": [
```

## Strategies

### 1. LLM Recommendations (`llm_recommendations`)

**Source**: `data/trades.json` (generated by AI analysis)

**Behavior**:
- Reads buy/sell recommendations from LLM
- **Auto-scales** to fit available buying power
- Rebalances entire portfolio to match recommendations
- Sells positions not in new recommendations

**Configuration**:
```json
{
  "auto_scale_to_buying_power": true,
  "max_buying_power_usage": 0.98,
  "trades_source": "data/trades.json"
}
```

### 2. Reddit 200 SMA on Leveraged ETF (`reddit_200sma_tqqq`)

**Source**: Rule-based, validated over 33 years

**Rules**:
- BUY when price > 200-day SMA
- SELL when price < 200-day SMA
- Uses all buying power when in position
- Works with TQQQ (3x QQQ), UPRO (3x SPY), etc.

**Configuration**:
```json
{
  "ticker": "TQQQ",
  "sma_period": 200,
  "max_position_size": 1.0
}
```

**Performance** (from backtests):
- 33-year CAGR: ~40%
- Sharpe: 0.686
- Max Drawdown: -58%

### 3. Dual Momentum (`dual_momentum`)

**Source**: Momentum rotation strategy

**Rules**:
- Calculate 6-month momentum for SPY, QQQ, GLD
- Hold asset with highest positive momentum
- Go to cash if all momentum is negative
- Rebalance monthly

**Configuration**:
```json
{
  "tickers": ["SPY", "QQQ", "GLD"],
  "lookback_period": 126,
  "rebalance_frequency": "monthly",
  "max_position_size": 1.0
}
```

## Margin & Buying Power

**Important**: Alpaca paper accounts support margin trading.

- **Negative cash** is normal and expected
- **Buying power** is what matters for purchases
- System uses `buying_power` not `cash` for trade sizing
- Max usage set to 98% of buying power for safety

Example:
```
Cash: -$100,363.50  ← Can be negative
Equity: $97,126.15
Buying Power: $194,252.30  ← This is what we can spend
```

## Auto-Scaling

When recommendations exceed buying power, the system auto-scales proportionally:

```
LLM recommends $370k in trades
Account has $100k buying power
→ Each trade scaled to 27% (100k / 370k)

XLU: $150k → $40.5k
IAU: $100k → $27.0k
MU: $75k → $20.3k
XLE: $25k → $6.8k
WDC: $20k → $5.4k
```

## Workflow Integration

The GitHub Actions workflow runs daily:

1. **Pull latest changes** (avoid merge conflicts)
2. Fetch latest price data
3. Run LLM analysis (generates `data/trades.json`)
4. **Execute multi-account trades**
5. Commit and push results

### Enable in Workflow

Edit `.github/workflows/daily_analysis.yml`:

```yaml
- name: Execute Multi-Account Trades
  env:
    ALPACA_API_KEY: ${{ secrets.ALPACA_API_KEY }}
    ALPACA_SECRET_KEY: ${{ secrets.ALPACA_SECRET_KEY }}
    ALPACA_ACCT2_API_KEY: ${{ secrets.ALPACA_ACCT2_API_KEY }}
    ALPACA_ACCT2_SECRET_KEY: ${{ secrets.ALPACA_ACCT2_SECRET_KEY }}
    # ... etc for all accounts
  run: |
    python src/execute_multi_account.py
```

## Safety Features

1. **Auto-scaling**: Never exceeds available buying power
2. **Dry-run mode**: Test without executing real trades
3. **Per-account enable/disable**: Turn accounts on/off in config
4. **Margin-aware**: Uses buying power, not cash
5. **Error logging**: All failures logged to `multi_account_logs/`
6. **Position validation**: Checks current positions before trading

3. **Register in** `src/strategies/momentum.py`

5. **Enable Account** when ready

```python
        # Your strategy logic
```

## Paper Trading vs Internal Tracking

**Paper Trading (Mode 3)**: Internal simulation with 7 virtual portfolios
- File: `src/execute_paper_trades.py`
- Data: `data/paper_trading/`
- Purpose: Test strategies without broker integration

**Multi-Account System**: Real Alpaca accounts with actual positions
- File: `src/execute_multi_account.py`
- Data: `data/multi_account_logs/`
- Purpose: Forward test with real broker, paper money

Both systems run independently and serve different purposes.

## Troubleshooting

### "No buying power available"

Check account status:
```bash
python src/fetch_alpaca_portfolio.py
```

Your account may be fully invested. Sell positions to free up capital.

### "Missing API keys"

Verify `.env` file has keys for account:
```bash
grep ALPACA_ACCT2 .env
```

### "Invalid strategy"

Check `config/accounts.json` has correct strategy name matching `src/strategies/momentum.py`.

### Merge Conflicts

Always commit/push local changes before the nightly workflow runs:
```bash
git add .
git commit -m "Local updates"
git push
```

Workflow now pulls before running to minimize conflicts.
